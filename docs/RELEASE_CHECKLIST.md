## Релиз в прод (чеклист)

### Перед деплоем
- **Секреты**
  - Убедиться, что `.env` и любые ключи/токены **не** лежат в Git.
  - Если что-то светилось в чатах/логах — **ротировать** (BOT_TOKEN, SESSION_JWT_SECRET, XUI/NocoBase, SSH ключи прокси).
- **ENV sanity**
  - `PUBLIC_BASE_URL` указывает на внешний домен (https), доступный Telegram.
  - `WEB_APP_URL` = `${PUBLIC_BASE_URL}/app/` (или актуальный путь).
  - `PAYMENT_WEBHOOK_SECRET` задан, если используешь `POST /api/webhooks/payment`.
  - Если задан `XUI_EXTRA_BASE_URLS`, то включить **secondary** (`XUI_SECONDARY_ENABLED=1`) и заполнить `XUI_SECONDARY_*`.
  - В проде выставить `CORS_ORIGIN` на конкретный домен мини‑аппа.
- **Хранилища**
  - Папка `data/` примонтирована в контейнер (см. `docker-compose.yml`), иначе привязки подписок потеряются при redeploy.

### После деплоя
- **Health**
  - `GET /health` отвечает `{ ok: true }`.
- **Telegram**
  - В логах нет бесконечных ошибок `setWebhook` (или задан `TELEGRAM_SKIP_WEBHOOK_SETUP=1` и вебхук настроен вручную).
- **Подписка RU+NL**
  - Новый пользователь → `Обновить ссылку (XUI)` → подписка содержит RU+NL.
  - Проверка апстримов: заголовок `x-sub-upstreams` на `GET /sub/xui/<token>`.
- **Платежи**
  - Telegram invoice link создаётся, сумма соответствует тарифу.
  - Вебхук провайдера при повторной доставке не выдаёт дублей (ответ `{ duplicate: true }`).
- **Баланс**
  - До первого пополнения почасовое списание не запускается.
  - После пополнения: списание идёт только когда XUI ACTIVE; при нуле — DISABLE, при пополнении — ENABLE.
