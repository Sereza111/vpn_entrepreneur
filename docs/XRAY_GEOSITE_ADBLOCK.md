# Блокировка рекламы в Xray: GeoSite `category-ads-all`

Подробная памятка для стека **VL** (подписка пользователей идёт из **3X-UI** / Xray). Настройка выполняется **на сервере панели**, а не в коде бота из этого репозитория.

## Зачем это нужно

- **Меньше рекламных запросов и трекеров** — часть доменов режется на стороне прокси до клиента.
- **Экономия трафика** — заметно на мобильных сетях.
- **Единые правила** для всех, кто ходит через ваш inbound, если правило включено в общий конфиг маршрутизации.

Это **не замена** блокировщику в браузере и **не шифрование** трафика отдельно — это **маршрутизация**: совпавший по списку доменный запрос не отправляется в «основной» outbound (часто направляют в `block`).

## Что такое GeoSite и откуда файл

**GeoSite** в экосистеме Xray/v2ray — это бинарный каталог доменов по **тегам** (категориям). В правилах маршрутизации пишут, например, `geosite:category-ads-all`, и Xray сопоставляет домен с этой категорией.

Проект **[runetfreedom/russia-blocked-geosite](https://github.com/runetfreedom/russia-blocked-geosite)** каждые несколько часов **пересобирает** файл **`geosite.dat`**, в который попадают в том числе:

- списки, связанные с блокировками и фильтрами (см. их README);
- категории из community-списков;
- **`geosite:category-ads-all`** — **рекламные домены** (в README проекта это отдельная категория).

Официальное описание и ссылки на скачивание: [README репозитория](https://github.com/runetfreedom/russia-blocked-geosite/blob/main/README.md).

### Прямые ссылки на `geosite.dat` (последний release)

- GitHub raw: `https://raw.githubusercontent.com/runetfreedom/russia-blocked-geosite/release/geosite.dat`
- Контрольная сумма (SHA256): `https://raw.githubusercontent.com/runetfreedom/russia-blocked-geosite/release/geosite.dat.sha256sum`
- Зеркала **JSDelivr** / **Fastly** — см. раздел «Скачать» в их README (удобно, если raw GitHub недоступен).

Перед обновлением на проде сверяйте хеш с `.sha256sum`, если критична целостность файла.

## Почему именно `geosite:category-ads-all`

- Одна **узкая цель**: реклама и сопутствующие списки, собранные в эту категорию upstream-проектом.
- **Не путать** с категориями вроде `geosite:ru-blocked` / `ru-blocked-all` — это про **другую задачу** (доступ к заблокированным в РФ ресурсам), другой объём списков и другие риски.

Если нужны и реклама, и отдельные списки — заводите **несколько правил** с разными `domain`/`geosite` и одним итоговым действием (например, `block`), и тестируйте по очереди.

## Минимальный чеклист (что сделать руками)

1. **Скачать** актуальный `geosite.dat` по ссылке выше (или с зеркала из README runetfreedom).
2. **Положить файл** туда, откуда Xray его читает. В **3X-UI** путь зависит от версии: часто это каталог данных Xray рядом с конфигом, либо загрузка через настройки «дополнительные файлы» / правка JSON — **откройте документацию вашей версии 3X-UI**.
3. В **`routing.rules`** (или эквивалент в UI) добавить правило **выше** общего `freedom`, чтобы оно сработало раньше:

   - **Условие**: домен в GeoSite, категория **`geosite:category-ads-all`**.
   - **Действие**: outbound с тегом **`block`** (или встроенный блокирующий outbound в вашей сборке).

   Пример **идеи** для сырого JSON Xray (не копируйте слепо — сверьтесь со своим полным конфигом):

   ```json
   {
     "type": "field",
     "domain": ["geosite:category-ads-all"],
     "outboundTag": "block"
   }
   ```

4. Убедиться, что в **`outbounds`** есть outbound с тегом **`block`** (тип `blackhole` или аналог).
5. **Применить** конфиг / **перезапустить** Xray (как требует панель).
6. **Проверка**: открыть сайт с явной рекламной сетью или посмотреть логи Xray; при поломке сайтов — временно отключить правило и сузить категорию.

## Риски и лучшие практики

| Риск | Что делать |
|------|------------|
| **Ложные срабатывания** | Редко домен из «рекламного» списка нужен для работы приложения. Если что-то сломалось — исключение через отдельное правило `direct` выше блока или отказ от слишком широкой категории. |
| **Большой размер списка** | Категория «вся реклама» может быть объёмной; на слабом CPU следите за задержкой и CPU. |
| **Обновления** | Upstream обновляет артефакты регулярно; имеет смысл **периодически** подтягивать новый `geosite.dat` по тому же URL или подписаться на [Releases](https://github.com/runetfreedom/russia-blocked-geosite/releases). |
| **Бэкап** | Перед правками **сохраните** текущий JSON конфига / экспорт из 3X-UI. |
| **DNS** | Не блокируйте без теста то, что может сломать резолвинг, если у вас нестандартная схема DNS в Xray. |

## Связь с продуктом VL

- Пользователи получают **подписку** с вашей **3X-UI**: изменения **маршрутизации на inbound** затрагивают всех клиентов этого inbound.
- **Сначала протестируйте** на своём устройстве; затем включайте для всех.
- Тексты в мини-аппе и **NocoBase** (`subscription_branding`) на правила Xray **не влияют** — только конфиг панели.
